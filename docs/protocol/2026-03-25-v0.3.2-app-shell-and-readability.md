# v0.3.2 Protocol — App Shell and Readability Pass

## Goal of this iteration

Address the first round of product-level feedback from a successful local test run.

The focus of this iteration is:

- reduce text noise
- improve button visibility
- improve readability of source selection and results
- add a basic app shell with a home page
- persist useful organizer defaults

## What changed

### App shell

- Added a basic shell with a **Home** page and an **Organize** page.
- The home page reflects the longer-term module structure without pretending the planned modules already exist.

### Organizer readability

- Source folders are now shown in a compact list with full paths available in tooltips and a details label.
- The result table was reworked to show shorter, more readable values instead of forcing full-path reading everywhere.
- Buttons now have explicit foreground styling so they remain visible.

### Organizer defaults

- The application now persists lightweight organizer defaults such as:
  - target folder
  - ExifTool path
  - target template
- If ExifTool can be auto-resolved, the GUI fills it automatically.

### Scope discipline

- Import sets were still **not** added in this iteration.
- Duplicate workflows, compare workflows, and localization switching were still deliberately deferred.

## Local validation focus

After pulling this version, validate the following:

1. buttons are clearly visible in all relevant places
2. the home page and organizer page both open correctly
3. the organizer page feels less text-heavy
4. source-folder items are easier to scan
5. the result table is easier to read than before
6. the ExifTool path is auto-filled or restored correctly
7. the target folder is restored correctly between launches

## Why this scope was chosen

Because the first successful technical run exposed the next bottleneck: **readability and product feel**.

Adding more major features at that point would have made the application broader but not better.
