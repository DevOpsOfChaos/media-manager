# v0.3.3 Protocol — Processing Feedback and Table Sizing

## Goal of this iteration

Improve real-world organizer usability during longer runs.

The focus of this iteration is:

- stronger feedback while processing is running
- a more stable run flow with less conflicting UI interaction
- more readable result-table sizing
- a stronger visual status column

## What changed

### Processing feedback

- The organizer now reports progress while it is running.
- The status bar and result-summary area are updated with short live messages such as scanning, processing, and finishing.

### Run stability

- Conflicting organizer controls are disabled during active processing.
- This reduces accidental input changes while a run is already in progress.

### Result table

- The status column is visually stronger and center-aligned.
- Result-table columns now resize more aggressively to their actual content.
- The window size was increased again so content sizing has more room.

### Scope discipline

- No import sets yet.
- No duplicate workflow yet.
- No comparison workflow yet.
- No localization switcher yet.

## Local validation focus

After pulling this version, validate the following:

1. the status bar updates while the organizer is processing files
2. the run summary text changes during the run instead of only after it finishes
3. organizer controls are disabled during an active run and return afterward
4. the status column is easier to notice in the result table
5. file, source, and target columns size themselves more readably than before
6. the wider window still feels usable on your screen

## Why this scope was chosen

Because a tool can be technically correct and still feel weak if the user cannot see progress or read the output comfortably.

This iteration improves the working feel of the organizer without drifting into new major feature areas.
