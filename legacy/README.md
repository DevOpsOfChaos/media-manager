# Legacy Area

This directory is reserved for files, notes, and subsystems that remain useful as reference but no longer define the active architecture of the repository.

## Important meaning

`legacy/` does **not** mean useless.

It means:

- keep for reference
- salvage only what still fits the new direction
- do not use as proof that the current architecture should stay the same

## Typical contents

Examples of material that may live here during the reset:

- older GUI-heavy experiments
- QML or desktop-UI prototypes
- notes tied to abandoned interaction models
- snapshots of superseded implementation directions

## Policy

New active development should target the rebuilt core-first structure.

This folder exists to reduce confusion, not to preserve confusion.
